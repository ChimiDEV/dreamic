Either

Overview#

A Either can be assumed to be either a Left (error) or Right (happy path) value. If a Either is observed as Left, a processing function has resulted in an error and you can easily handle it without having to fear about crashing process, because of a thrown error.

The either monad abstracts away annoying try/catch blocks. While functions applying to Right will work as excepted, a Left simply returns itself without invoking any unsafe operation. Since the error is carried on you can inspect the problem at the end of the process pipeline, while still failing as fast as possible.

import { Either } from '@chimidev/dreamic';
Either.of<string>('12').map(parseInt); // Either.Right(12)
Either.of<string>('unparsable').map(parseInt); // Either.Left(ParserError) - No error is thrown
